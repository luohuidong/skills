# UpGrade Dependencies Skill

1. 检查用户指定的依赖的版本是否过时。
   1. 如果依赖没有过时，则通知用户当前依赖已经是最新版本，无需做进一步的操作。
   2. 如果依赖过时，则继续下一步检查升级的影响。
2. 检查升级是否会引入破坏性更改。同时也查看一下跟依赖相关的其他依赖是否也需要升级。资料可以尝试以下途径获取：
   1. 使用 web search 来查找相关的升级日志、社区讨论和文档，以评估升级的风险和影响。
   2. 使用 github cli `gh` 命令来查看相关仓库的 release 记录来查看版本变更内容。
   3. 查看官方文档看看是否有 migration guide 来评估升级的风险和影响。
3. 如果没有破坏性更改，则将依赖升级到最新的版本。
4. 如果有破坏性更改，则提供升级建议和替代方案，帮助用户做出决策。
5. 升级完成后，运行测试以确保项目的稳定性和兼容性没有受到影响。
6. 最后，生成升级报告，记录升级过程中的变化和结果，供用户参考。
